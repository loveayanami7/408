## 性能指标
1. 数据传输速率 = 数据宽度/存取周期
2. 存取时间：启动一次存储器操作到完成该操作所用时间，分**读出时间**和**写入时间**
3. 存取周期：存储器进行一次完整的读/写操作所需的全部时间，即两次独立操作之间的最小时间间隔
4. 存取周期 > 存取时间![[Pasted image 20260513140258.png]]
## 主存
### SRAM
* 存储元：双稳态触发器（六晶体管mos）
* 非破坏性读出，无须再生
* 存取速度快，但集成度低，功耗大，价格高
### DRAM🌟
#### 工作原理
* 存储元：电容
* 电荷会因为漏电而逐渐丢失，必须定期刷新
* 破坏性读出，读取后需要再生
* 存取速度较SRAM慢，但集成度高，功耗低，价格便宜，容量大
#### 关键技术
1. 地址引脚复用
	行列地址共用引脚，分时传输
2. 刷新机制
	* 按行刷新
	* 行地址缓冲器与刷新计数器
	* 地址引脚数量由行列地址中较大者决定
	* 行地址位数 >= 列地址位数
3. 缓存机制与突发传输
	* DRAM芯片内部集成行缓冲器，SRAM实现
	* 突发传输定义

## ROM
1. MROM：掩模式，用户不可更改
2. PROM：允许用户进行一次性编程
3. EPROM：可擦除可编程，擦除须紫外线照射整片芯片，擦写次数有限，写入速度慢
4. Flash：闪存，由EPROM发展来，非易失，电擦除在线重写，读取速度接近RAM，写入显著较慢
## 多模块存储器
### 连续编址
* 地址高位为体号，低位为体内地址
* 总是一个体的存储单元全部读取完才进入下一个体，不能并行访问，吞吐量小
* 体0地址0 -> 体0地址1 -> ... -> 体1地址0 -> ...
### 低位交叉编址
* 地址高位为体内地址，低位为体号
* 体号 = 地址 % m（体内存储单元数）
* 连续地址的数据被分配到不同的存储体中，提高吞吐量
* 体0地址0 -> 体1地址0 -> 体2地址0 -> ... -> 体0地址1 -> ...
#### 轮流启动
* 计算为满足轮流启动，存储器交叉模块数应该有多少 -> m >= T/r
* 计算连续取m个字所需时间 -> t = T + (m-1)r
* 交叉存储器访存冲突分析 -> 相邻的m次访存地址出现在同一个体内则发生冲突
#### 同时启动
* 注意提取题干信息 -> 交叉编址 -> 总线宽度 = 所有存储模块一次并行读/写的总位数 -> 采用同时启动
* 还要注意数据的放置位置，一个存取周期可能会读出无效的数据 -> 了解编址方式

### 与主存的映射方式
#### 不同映射方式对应的地址格式不同
* 直接映射 -> 根据主存地址确定放在cache哪一行，块号固定
	* Tag+块号+块内地址
* 全相联映射 -> 随机放在cache中某一行
	* Tag+块内地址
* 组相连映射 -> n个cache块为一组，即为n路组相连，根据主存地址确定组号，随机放在组内某一行
	* Tag+组号+块内地址
#### cache访问冲突分析
* 多个主存块映射到同一个cache块就发生冲突
* 不同映射方式会影响冲突率 -> 冲突率：直接>组相连>全相连
#### 比较器
* 一般每个cache行设置一个比较器
* 比较器位数 = Tag字段位数
* 直接映射每块只对应唯一cache行，因此只需设置1个比较器
* n路组相联映射每块要与一组中的n行并行比较，因此要设置n个比较器
